## 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/arch3rpro/trae-proxy.git
cd trae-proxy

# 2. 启动服务（Docker 方式）
docker-compose up -d

# 3. 查看日志
docker-compose logs -f
```

## 环境要求

### 服务器端
- Docker 和 Docker Compose
- 或 Python 3.9+
- OpenSSL（用于生成证书）

### 客户端
- 管理员权限（修改 hosts 文件、安装证书）
- 支持自定义模型的 IDE（如 Trae、VS Code 等）

## 安装步骤

### 方式一：Docker（推荐）

```bash
# 1. 确保 Docker Desktop 正在运行
docker --version
docker ps

# 2. 启动容器
docker-compose up -d

# 3. 验证容器状态
docker-compose ps

# 4. 查看日志
docker-compose logs -f
```

### 方式二：手动部署

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 生成 SSL 证书
python generate_certs.py

# 3. 启动代理服务器（需要管理员权限，监听 443 端口）
python trae_proxy.py
```

## 配置说明

### 配置文件结构

`config.yaml` 是主配置文件：

```yaml
# Trae-Proxy 配置文件
# 代理域名配置
domain: api.openai.com

# 后端 API 配置列表
apis:
  - name: "模型名称"
    endpoint: "https://api.example.com"  # ⚠️ 不要带 /v1 后缀
    custom_model_id: "模型 ID"
    target_model_id: "目标模型 ID"
    stream_mode: null
    active: true
# 代理服务器配置
server:
  port: 443
  debug: true
```

### ⚠️ 重要：endpoint 格式

**错误写法：**
```yaml
endpoint: "https://api.example.com/v1"  # ❌ 会导致 /v1/v1/ 重复
```

**正确写法：**
```yaml
endpoint: "https://api.example.com"  # ✅ 代理会自动添加 /v1
```

### 配置示例

```yaml
apis:
  - name: "gpt-5.3-codex"
    endpoint: "https://api.giiip.com"
    custom_model_id: "gpt-5.3-codex"
    target_model_id: "gpt-5.3-codex"
    active: true
  - name: "qwen3.5-plus"
    endpoint: "https://coding.dashscope.aliyuncs.com"
    custom_model_id: "qwen3.5-plus"
    target_model_id: "qwen3.5-plus"
    active: true
```

### 修改配置后重启

```bash
docker-compose restart
```

## 客户端配置

### 步骤 1：修改 hosts 文件

**Windows（管理员身份 PowerShell）：**
```powershell
Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" -Value "`n127.0.0.1 api.openai.com # Trae-Proxy" -Force
```

**验证：**
```bash
ping api.openai.com
# 应返回 127.0.0.1
```

### 步骤 2：安装 CA 证书

证书位置：`ca/ca.crt`

**Windows 安装步骤：**
1. 双击 `ca/ca.crt` 文件
2. 点击"安装证书"
3. 选择"本地计算机"
4. 选择"将所有证书放入下列存储" → 浏览 → 选择"受信任的根证书颁发机构"
5. 完成安装

**PowerShell 命令安装（管理员）：**
```powershell
Import-Certificate -FilePath "ca\ca.crt" -CertStoreLocation Cert:\LocalMachine\Root
```

### 步骤 3：配置 IDE

以 Trae IDE 为例：

1. 打开 Trae → 设置 → 模型
2. 点击"添加模型"
3. 填写：
   - **服务商**: OpenAI
   - **模型 ID**: 配置文件中定义的 `custom_model_id`
   - **API 密钥**: 对应后端 API 的 Key
4. 点击"添加模型"

## 常见问题

### Q1: 添加模型失败，提示"模型请求失败"

**可能原因及解决：**

| 原因 | 检查方法 | 解决方案 |
|------|----------|----------|
| hosts 未生效 | `ping api.openai.com` | 以管理员身份重新修改 hosts |
| 证书未安装 | curl 测试报错 | 重新安装 CA 证书到"受信任的根" |
| 代理服务未运行 | `docker-compose ps` | 重启容器 `docker-compose restart` |
| Clash 等代理干扰 | 关闭代理测试 | 在 Clash 中添加直连规则 |

### Q2: 开启 Clash 后无法连接

**问题原因：** Clash 的 TUN 模式接管了系统流量，导致 `api.openai.com` 请求被转发到代理而不是本地。

**解决方案：** 在 Clash Verge 中添加规则：

1. 点击"全局扩展覆写配置" → Merge
2. 添加：
```yaml
rules:
  - DOMAIN,api.openai.com,DIRECT
```
3. 保存并更新配置

### Q3: 404 Not Found 错误

**错误信息：**
```
404 Not Found for url: https://xxx/v1/v1/chat/completions
```

**原因：** endpoint 配置重复了 `/v1` 后缀

**解决：** 修改 `config.yaml`，去掉 endpoint 末尾的 `/v1`：
```yaml
# ❌ 错误
endpoint: "https://coding.dashscope.aliyuncs.com/v1"

# ✅ 正确
endpoint: "https://coding.dashscope.aliyuncs.com"
```

### Q4: SSL 证书验证失败

**错误信息：**
```
CRYPT_E_NO_REVOCATION_CHECK (0x80092012)
```

**解决方案：**

1. 禁用证书吊销检查（管理员 PowerShell）：
```powershell
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\SystemCertificates\Root\Certificates" -Name "NoRevocationCheck" -Value 1 -Type DWORD -Force
```

2. 重启电脑

### Q5: 代理日志中没有请求记录

**问题：** Trae 的请求没有到达代理服务器

**可能原因：**
- Electron 应用缓存 DNS
- 系统代理设置干扰

**解决方案：**
1. 完全重启 Trae（任务管理器确认进程已结束）
2. 刷新 DNS：`ipconfig /flushdns`
3. 检查 Trae 是否启用了内置代理设置

## 故障排查

### 测试代理是否可访问

```bash
# 测试本地代理
curl -k https://127.0.0.1/v1/models

# 测试 hosts 解析后
curl -k https://api.openai.com/v1/models

# 带 API Key 测试
curl -k https://api.openai.com/v1/models -H "Authorization: Bearer your-api-key"
```

**预期返回：**
```json
{
  "data": [
    {"id": "gpt-5.3-codex", "object": "model", "owned_by": "trae-proxy"},
    {"id": "gpt-5.4", "object": "model", "owned_by": "trae-proxy"}
  ],
  "object": "list"
}
```

### 查看容器日志

```bash
# 查看最近日志
docker-compose logs --tail 50

# 实时跟踪日志
docker-compose logs -f
```

### 检查容器状态

```bash
docker-compose ps
docker-compose top
```

### 验证配置已加载

```bash
docker-compose logs 2>&1 | grep "多后端配置"
```

应显示类似：
```
多后端配置已启用，共 5 个 API 配置，5 个激活
```

## 项目结构

```
trae-proxy/
├── trae_proxy.py          # 主代理服务器
├── trae_proxy_cli.py      # 命令行管理工具
├── generate_certs.py      # 证书生成工具
├── config.yaml            # 配置文件
├── docker-compose.yml     # Docker 部署配置
├── requirements.txt       # Python 依赖
└── ca/                    # 证书和密钥目录
    ├── ca.crt             # CA 证书
    ├── ca.key             # CA 私钥
    ├── api.openai.com.crt # 服务器证书
    └── api.openai.com.key # 服务器私钥
```
